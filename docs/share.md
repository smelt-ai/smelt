# smelt —— Mac 上的 AI coding 驾驶舱

*内部技术分享稿 · 2026-07-09*

> 一个基于 GPUI 的桌面工作台，内嵌真终端，专为「同时指挥多个 Claude Code / Codex agent 干活」设计——多项目 × 多标签，会话状态一目了然。

**状态**：working prototype · 持续迭代中
**语言**：Rust 2021 ｜ **GUI**：GPUI ｜ **规模**：~10,255 行 ｜ **二进制**：`workspace` + `smeltd`

---

## 01 · 为什么做这个

### 编辑工具进化史：为什么终端会成为下一个入口

1. **IDE 时代** —— IDEA、VS Code 这类图形化编辑器，靠语法高亮、代码补全把「写代码」标准化，代价是内存占用大、启动慢、扩展生态越叠越臃肿。
2. **AI 插件化时代** —— Copilot、Tabnine 等以插件形式接入已有编辑器：光标后拉出一行灰色的补全建议，或者在右侧拽出一个窄侧边栏对话。这本质上是一种折中——**编辑器的核心交互逻辑没有变**，人依然是敲键盘的那个苦力，AI 只是个被动响应、等你问了才说话的侧边栏挂件。
3. **AI 原生编辑器时代** —— Cursor 这类产品把 AI 从插件位置挪进编辑器内核：inline diff、多文件改动、agent 模式，AI 的话语权明显变大了，但用户打开的依然是一个「编辑器」，本质上还是在盯着一份代码改。
4. **AGI 时代：主驾驶的交接** —— 早期的 AI 插件是 IDE 里的「副驾驶」，回答你问、补全你打；而当 agent 真正能独立跑完一整个任务（读代码、改代码、跑测试、提交），人的角色就该从「打字的人」变成「看导航、下指令的人」。这时候需要的不再是一个更聪明的编辑器，而是一个能同时看住好几个正在跑的 agent 的**驾驶舱**——这正是 smelt 想站的位置：不做第 3 阶段那个更聪明的编辑器，直接跳到第 4 阶段，把终端（agent 真正干活的地方）变成主战场。

### 从「蒸馏 CLI」到「驾驶舱」的一次换赛道

smelt 最早不是这个方向——它起步于一条 **instincts 蒸馏** 链路（`observe`/`digest`/`merge`）外加一个飞书客户端。用下来发现真正每天磨人的问题不在那条链路上，而在更基础的地方：**同时开好几个 Claude Code 会话，各自散在系统终端的一堆窗口 / tab 里**，切来切去找不到谁在等我输入、谁跑完了、谁还在 thinking；GUI 一关或者崩了，终端里的会话也就没了。

于是做了一次明确的收敛（`afa4a90` "移除 instincts 蒸馏链路与飞书客户端，聚焦 AI 驾驶舱方向"）：仓库里现在只剩两个二进制——`workspace`（GUI 主程序）和 `smeltd`（终端持久化守护），目标很单一：**把「多项目 × 多个 agent 终端」这件事，做成一个专用的驾驶舱外壳**，而不是在系统终端里硬凑。

## 03 · 桌面宠物：不只是彩蛋

一句话：独立的透明置顶浮窗，陪着你，跟主工作台的终端 / git / 文件树完全解耦——关掉整个宠物模块不影响任何驾驶舱功能。

### 几个特色

1. **纯手绘，零图片资源** —— 宠物本体是 GPUI 图元手绘出来的，不依赖任何 png/svg 素材；20fps 定时器驱动待机动画：呼吸起伏、上下浮动、眨眼。
2. **全局透明穿透浮窗** —— 单开一个 `NSPanel`（`WindowKind::PopUp`）：无边框、透明背景、浮在所有窗口之上、跟随所有 Space、不抢焦点；宠物本体之外的透明区域整体点击穿透，不挡下层操作。
3. **眼神跟随鼠标** —— 用 `[NSEvent mouseLocation]` 取全局光标坐标（不是窗口内坐标——宠物窗口大半透明，`window.mouse_position()` 光标不在窗内时就失效），实时算瞳孔偏移，视线始终追着光标走。
4. **情绪 / 状态感知** —— 维护一个随交互衰减的「忙碌值」（指数衰减，约 35s 半衰期）驱动情绪状态；10 分钟无鼠标移动判定 AFK；切换前台 app 后带 75s 冷却地评论一句；空闲 160s 主动搭话。
5. **可选接 LLM 大脑** —— 默认用固定台词库轮换（点一下蹦一下、换句话），开关打开后台词改成调用 OpenAI 兼容协议的模型生成，喂给它当前前台 app 名当上下文。
6. **跨窗口消息投递** —— 主程序任意位置调 `push_pet_message` 往全局单例信箱塞一条消息，宠物气泡就会把它「说」出来（比如通知类事件）。

### 拖拽这块的实现，踩过一次坑

最早直接用 AppKit 的 `performWindowDragWithEvent` 做拖拽——省事，但它是**模态调用**，拖拽过程中会冻结 App 自身的动画循环，宠物拖着走的时候待机动画就卡住了。改成手动实现：`mouse_down` 记录锚点（鼠标位置 + 窗口原点），松开前用一个独立的 **60fps 高频定时器**轮询全局鼠标坐标，算出偏移量后直接 `setFrameOrigin` 挪窗口——不挂在 GPUI 原始 mouse move 事件上，因为那个事件频率不稳，跟手感很差。这个高频时钟和驱动待机动画的 20fps 慢时钟是两条独立的定时器：不拖拽时高频时钟立刻判负返回，开销可忽略，不会拖累空闲功耗。

过程中还带出一个隐蔽 bug：GPUI 的 `HitboxId` 是每帧单调递增分配的，`mouse_down` 时 capture 的 hitbox 隔一帧就失效，导致 `on_mouse_up` 判断"是不是这次拖拽在松手"会**静默失灵**——表现为松手后宠物有时会诡异地继续跟着鼠标漂移。改成监听 window 级别的 `on_mouse_event` 而不依赖 hitbox 生命周期，规避掉了。（装饰性的挤压拉伸/滚球动画试了很多轮效果都不理想，最后按要求整体去掉，只留了抓取光标和语音气泡。）

配置（显示开关、通知播报、颜色、大小）存在 `~/.smelt/pet.json`，跨窗口走全局单例共享，由主窗口设置面板编辑。

## 04 · 技术架构亮点

### 选型服从一个目标：真终端 + 不卡

| 层 | 选型 | 说明 |
|---|---|---|
| GUI 框架 | GPUI（zed-industries/zed） | GPU 加速，Metal 渲染，锁定固定 commit |
| 组件库 | gpui-component（longbridge） | Dock / Sidebar / Settings 等，同一 gpui 生态 |
| 终端后端 | portable-pty | 起 `$SHELL` 子进程并拿到 PTY |
| 终端状态机 | alacritty_terminal | 只用它的 ANSI 解析 + 网格模型，不用自带 event loop |
| 定时器 | smol | 驱动终端网格快照重绘 |

**关键决策**

- **runtime_shaders —— 免装完整 Xcode**：Metal 着色器改到运行时编译，编译期不调 `xcrun metal`，只装 Command Line Tools 就能构建，降低团队里其他人上手的门槛。
- **portable-pty + alacritty「仅状态机」**：不用 alacritty 自带的 event loop，只借它的 ANSI 解析和网格模型，自己接 GPUI 的渲染循环——集成更干净，也更好控制重绘时机。
- **IME 走 EntityInputHandler**：中文等合成输入不经普通按键事件，必须实现输入处理器并在 paint 阶段注册；同一个 canvas 顺便记录网格原点，供鼠标坐标换算复用。
- **整行 StyledText 渲染**：放弃逐格定宽 span，改成整行一次性成形、多 TextRun 上色——解决了拖选时的子像素抖动和宽度截断，比逐格渲染更快。

### smeltd：持久化守护的三条不变量

```
workspace（GUI） ──attach / 输入──▶ smeltd（守护） ──PTY 管理──▶ shell / claude
可随时退出 / 崩溃                按会话 id 落盘，字节流 + 重放      真实子进程，GUI 生死与它无关
```

1. 会话 **id 创建即落盘**，不等首次写入，避免「刚建就丢」
2. **attach 先回报 PTY 真实尺寸**，GUI 按它建终端，再收重放字节，杜绝尺寸协商时序错乱导致的花屏
3. 同一会话的读写走**同一把锁串行**，防止并发 attach 把输出交叉打乱

### 一次真实的性能排查

终端 30ms 定时器以前无条件触发重绘，哪怕 shell 完全空闲也按 **33 次/秒**的频率重画。改用 alacritty 的 `Term::damage()` 判断内容是否真变化后，空闲终端 5 秒内 `render()` 从理论值降到 **8 次**。排查过程中顺带抓到一个真 bug——`damage_cursor()` 曾无条件标脏光标格，必须排除「仅光标那一格」的脏区才算数，现有回归测试 `damage_gate_tests` 覆盖。

另外一个小知识点：**桌面宠物的透明无边框浮窗**是靠绕开 GPUI 实现的——GPUI 在 macOS 上只会建带圆角边框 + 阴影的 Titled 窗口，宠物浮窗直接拿到底层 `NSWindow` 用 `objc` 手改 `styleMask` / 阴影 / 背景色做出来的。
